# 📋 ISO 27001 Ek-A Kontrolleri Tam Listesi (114 Kontrol)

Bu tablo, ISO/IEC 27001:2013 standardında yer alan tüm kontrol maddelerinin hızlı bir referans listesidir.

| ID | Kontrol Adı | Kısa Açıklama |
| :--- | :--- | :--- |
| **A.5.1.1** | Bilgi güvenliği politikaları | Yönetim tarafından onaylanmış yazılı kurallar seti. |
| **A.5.1.2** | Bilgi güvenliği politikalarının gözden geçirilmesi | Politikaların güncelliğinin kontrolü. |
| **A.6.1.1** | Bilgi güvenliği rolleri ve sorumlulukları | Kimin neyden sorumlu olduğunun netleşmesi. |
| **A.6.1.2** | Görevler ayrılığı | Hata ve suistimali önlemek için kritik yetkilerin bölünmesi. |
| **A.6.1.3** | İlgili makamlarla iletişim | BTK, KVKK gibi kurumlarla temas noktalarının belirlenmesi. |
| **A.7.2.2** | Bilgi güvenliği farkındalığı, eğitim ve öğretim | Personelin güvenlik bilincinin artırılması. |
| **A.8.1.1** | Varlık envanteri | Tüm bilgi varlıklarının listelenmesi. |
| **A.8.2.1** | Bilginin sınıflandırılması | Gizli, Hizmete Özel, Genel gibi etiketleme. |
| **A.9.1.1** | Erişim kontrol politikası | Kimin nereye erişebileceğinin kuralları. |
| **A.11.1.1** | Fiziksel güvenlik sınırı | Kartlı geçiş, kapı, pencere güvenliği. |
| **A.12.1.1** | Belgelenmiş işletim prosedürleri | Sistemlerin nasıl yönetileceğine dair kılavuzlar. |
| **A.12.4.1** | Olay kaydı tutma (Logging) | Sistem loglarının (ERİŞİM, HATA) kaydedilmesi. |
| **A.14.2.1** | Güvenli geliştirme politikası | Yazılım yazarken güvenlik standartlarına uyum. |
| **A.17.1.1** | Bilgi güvenliği sürekliliğinin planlanması | Afet anında güvenliğin nasıl sağlanacağı. |
| **A.18.1.1** | Uygulanabilir yasaların tespiti | Yasalar ve regülasyonlara (KVKK vb.) uyum. |

> [!NOTE]
> Bu tablo sadece kritik örnekleri içermektedir. Tam 114 kontrolün listesi ve detayları için kurumunuzun satın aldığı resmi ISO standardı dokümanına başvurmanız önerilir.
