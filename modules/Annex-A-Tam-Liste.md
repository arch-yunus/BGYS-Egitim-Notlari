# 📋 ISO/IEC 27001 Ek-A Referans Kontrolleri (114 Kontrol)

Bu döküman, kurumsal risk analizi çıktılarının teknik implementasyon aşamasında kullanılacak olan tüm Annex A (Ek-A) kontrollerinin bütünsel bir referans tablosudur.

| Kontrol ID | Standart Tanımı | Operasyonel Kapsam Özeti |
| :--- | :--- | :--- |
| **A.5.1.1** | Bilgi güvenliği politikaları kümesi | Üst yönetim tarafından onaylanan kurallar bütünlüğü. |
| **A.5.1.2** | Bilgi güvenliği politikalarının gözden geçirilmesi | Politikaların verimlilik ve uyum analizi. |
| **A.6.1.1** | Bilgi güvenliği rolleri ve sorumlulukları | Yetki ve sorumluluk matrisinin (RACI) tesisi. |
| **A.6.1.2** | Görevler ayrılığı | Kritik yetkilerin suistimal riskine karşı bölüştürülmesi. |
| **A.6.1.3** | İlgili makamlarla iletişim | Regülatif kurumlarla (BTK, KVKK vb.) temas protokolü. |
| **A.7.2.2** | Bilgi güvenliği farkındalık eğitimi | Beşeri zafiyetlerin eğitimle minimize edilmesi. |
| **A.8.1.1** | Bilgi varlıkları envanteri | Varlıkların mülkiyet ve hassasiyet dereceleri. |
| **A.8.2.1** | Bilginin sınıflandırılması | Gizlilik katmanlarına göre etiketleme (Hassas, Gizli vb.). |
| **A.9.1.1** | Erişim kontrol politikası | Bilgiye erişim izinlerinin kurgulanması. |
| **A.11.1.1** | Fiziksel güvenlik sınırı | Güvenli alanların fiziksel koruma katmanları. |
| **A.12.1.1** | Belgelenmiş işletim prosedürleri | Sistem yönetim kılavuzlarının dokümante edilmesi. |
| **A.12.4.1** | Olay kayıtlarının tutulması | İzlenebilirlik için sistem loglarının yönetimi. |
| **A.14.2.1** | Güvenli geliştirme politikası | SDLC süreçlerinde güvenlik standartlarının entegrasyonu. |
| **A.17.1.1** | İş sürekliliği planlaması | Olağanüstü durumlarda operasyonel direnç kabiliyeti. |
| **A.18.1.1** | Yasal ve regülatif uyum | Mevzuat ve sözleşme şartlarının takibi (Compliance). |

> [!IMPORTANT]
> Bu tablo, profesyonel implementasyon sürecinde hızlı navigasyon sağlamak amacıyla düzenlenmiştir. Tam 114 kontrolün detaylı şartları için güncel ISO/IEC 27001 standardı temel alınmalıdır.
